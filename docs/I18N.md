# Internationalisation

The popup ships with three locales: English (`en`), French (`fr`), and Arabic (`ar`). Each user picks a locale from the language selector in the header; the choice persists in `chrome.storage.local.uiLanguage`.

## The translation table

`shared/i18n.js` exports a single object keyed by locale:

```js
export const I18N = {
  en: { appTitle: "Qur'an & Sunnah Companion", quran: "Qur'an", play: "Play", ... },
  fr: { appTitle: "Compagnon du Coran et de la Sunnah", quran: "Coran", play: "Lire", ... },
  ar: { appTitle: "رفيق القرآن والسنة", quran: "القرآن", play: "تشغيل", ... }
};

export function t(key) {
  return (I18N[currentLang] && I18N[currentLang][key]) || key;
}
```

Every key in `en` must exist in `fr` and `ar`. Missing keys fall back to the key string itself, which is visible in the UI.

## Religious content is read-only

The dhikr Arabic, the dhikr transliteration, the dhikr English, the dhikr French, the AR/FR reward translations, and every fetched hadith are religious content. Nobody on the codebase rewrites them — not maintainers and not AI assistants. They live in `shared/dhikr.js` (the dhikr fields and reward maps) and come from third-party APIs (the hadith strings).

If something looks off, open an issue with a primary-source citation and let the maintainer reconcile against an authenticated collection.

UI strings (button labels, error messages, headers, the language selector) are translatable freely. They are not religious content.

The clear-button affordances next to the surah and reciter pickers each carry their own translation key:

- `clearReciter` — `"Clear"` (en), `"Effacer"` (fr), `"مسح"` (ar). Bound to the reciter picker's inline clear icon via `aria-label`.
- `clearSurah` — `"Clear"` (en), `"Effacer"` (fr), `"مسح"` (ar). Bound to the surah picker's inline clear icon via `aria-label`. Added in 2.1.1 alongside the combobox redesign.
- `clearSurahButton` — `"Clear surah"` (en), `"Effacer la sourate"` (fr), `"مسح السورة"` (ar). The visible label and `aria-label` for the quick-clear button under the sleep timer. Added in 2.1.2.
- `clearReciterButton` — `"Clear reciter"` (en), `"Effacer le récitateur"` (fr), `"مسح القارئ"` (ar). The visible label and `aria-label` for the second quick-clear button. Added in 2.1.2.

The "Stop playing" banner that appears when the popup's selection differs from the audio playing in the background uses two keys:

- `stopPlaying` — `"Stop playing"` (en), `"Arrêter la lecture"` (fr), `"إيقاف التشغيل"` (ar). Bound to the banner's discard button via `aria-label`.
- `playingBannerLabel` — `"Stop playing: {surah} · {reciter}"` (en), `"Arrêter la lecture : {surah} · {reciter}"` (fr), `"إيقاف التشغيل: {surah} · {reciter}"` (ar). The banner text. `{surah}` and `{reciter}` substitute at render time. Added in 2.1.2.

## RTL handling

When the user picks Arabic, `applyLanguage()` sets:

- `document.documentElement.lang = 'ar'`
- `document.body.dir = 'rtl'`

CSS responds via `body[dir="rtl"]` selectors:

- Body font becomes `var(--font-arabic), var(--font-body)` so Latin glyphs in Arabic strings (English numerals in inputs, technical punctuation) still render correctly.
- Header and card-controls flex direction reverses with `flex-direction: row-reverse`.
- The audio time display (`0:23 / 4:51`) overrides with `direction: ltr` so the colon-separated timestamp keeps its natural reading order.
- Number inputs (interval seconds) use Western Arabic numerals, not Eastern. This matches the validator and avoids parsing surprises.

The dhikr text container has `dir="auto"` so mixed Arabic + Latin segments render in their natural directions.

## Adding a locale

1. Add the locale to `shared/i18n.js`. Match every key from the `en` block.
2. Add the locale to the `<select>` options in `chrome/popup/index.html` and `firefox/popup/index.html`.
3. If the locale is RTL (Hebrew, Persian, Urdu, etc.), update `applyLanguage()` to set `dir = 'rtl'` for the new locale and verify the CSS selectors `body[dir="rtl"]` cover the new locale's behaviour.
4. Optionally extend `shared/hadith.js` with a fetcher for the locale. The jsdelivr `hadith-api` source ships translations in many languages; pick editions and reasonable counts.
5. Run `npm run sync` and reload both builds. Test the language switch round-trips and refreshes the hadith.

## Locales we will not add

- Locales that require LLM-translated religious content. The constraint is the same as the read-only rule. We accept locales where authenticated translations exist for the dhikr collection and where hadith APIs serve verified text. We do not accept locales where someone has to translate the dhikr themselves.

## Browser UI language detection

On first launch, `initLanguage()` checks the user's `navigator.language` (or `chrome.i18n.getUILanguage()`). If the prefix matches a supported locale, that locale is used. Otherwise English is the fallback. The user can override the auto-detection with the language selector at any time.
